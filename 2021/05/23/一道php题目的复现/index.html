那么这就涉及php数组下标溢出。在php中int型变量的范围为2^31也就是2147483647。当我们设置user类中的count值为2147483646时。再对其加1就会产生报错Warning: Cannot add element to the array as the next element is already occupied 当设置count为2147483647时则会产生上界溢出，不会产生任何警告。下标被设置为-9223372036854775808。但是我在测试时发现php版本为5.5.12时并未观察到警告出现。这样一来$count[++$user->count]=1;语句执行后$user->count会变成int上界2147483647。而后面的if($count[]=1)也会返回false进而达到绕过的目的

所以我们构造payload序列化串O:4:"User":1:{s:5:"count";i:9223372036854775806;}这里值得注意的是需要对其进行urlencode，所以最后填到cookie里应该是这样的

在蚁剑中也有通过攻击 PHP-FPM 绕过 Disable_Functions 的插件啊，但当时为什么就用不了呢？原来他蚁剑的 Payload 中用到的 fsockopen()，而 fsockopen() 被 Disable_Functions 给禁了，并且蚁剑的插件中没有 PHP-FPM 地址为 127.0.0.1:9001 的选项，所以自然就成功不了：

但是在 PHP 中除了 fsockopen() 这个函数外，还有一个 pfsockopen() 二者几乎没有什么区别，在 fsockopen() 被禁用后我们可以用 pfsockopen() 进行代替。我们可以对蚁剑的 Payload 进行修改，即将 Payload 所有的 fsockopen() 替换为 pfsockopen()。需要替换的文件有以下几个：

  • \antData\plugins\as_bypass_php_disable_functions-master\payload.js
  • \antData\plugins\as_bypass_php_disable_functions-master\core\php_fpm\index.js

            $nvpair = chr($nlen);

                $buf=fread($this->_sock, $resp['paddingLength']);
            }
            return $resp;
        } else {
            return false;
        }
