简单看一下就可以得知以get形式访问根目录会得到一个图片上传界面，以POST形式访问根目录就会把图片上传。还有个image路由，我们稍后再看。

先看一下上传的过滤

IndexController.php

class IndexController extends Controller
{
    public function fileUpload(Request $req)
    {
        $allowed_extension = "png";
        $extension = $req->file('file')->clientExtension();
        if($extension === $allowed_extension && $req->file('file')->getSize() < 204800)
        {
            $content = $req->file('file')->get();
            if (preg_match("/<\?|php|HALT\_COMPILER/i", $content )){
                $error = 'Don\'t do that, please';
                return back()
                    ->withErrors($error);
            }else {
                $fileName = \md5(time()) . '.png';
                $path = $req->file('file')->storePubliclyAs('uploads', $fileName);
                echo "path: $path";
                return back()
                    ->with('success', 'File has been uploaded.')
                    ->with('file', $path);
            }
        } else{
            $error = 'Don\'t do that, please';
            return back()
                ->withErrors($error);
        }


    }
}

可以看到，文件上传的拓展名被钉死为png。同时文件大小不能太大，这个限制条件可以忽略。然后再看把上传的文件进行一个处理用get()方法获取文件内容。其本质就是一个file_get_contents()如果在文件内容找到preg_match("/<\?|php|HALT\_COMPILER/i", $content )就会报错。否则存储文件，并回显储存地址。再看那个神秘的/image路由它接受GET方法传入的参数，它会尝试重新命名上传的png然后对其进行

这样的一个调用，其中compressImg()会调用imgcompress类的_openImage方法。让我们看一下这个方法的实现

可以看到很明显的getimagesize函数。因而考虑反序列化phar来RCE。同时结合框架是Laravel猜想。那么目前的主要问题就是如何成功构造这样一个phar来进行上传。那我们自然考虑到常用的压缩法。gzip压缩phar再把压缩文件尾缀改为png再上传即可绕过waf。再通过路由/image进行触发即可达成RCE。此处坑点一方面在于上传路径为phar://../storage/app/uploads/XXXX.png 这是因为Laravel默认目录处于public下，所以需要进行跳转。另一点就是buu可能改变了Laravel的版本。通过

php .\artisan --version

当PHP函数中底层调用了php_stream_open_wrapper,都能够被phar的组件用来进行解析Phar文件(文件内容需要包含__HALT_COMPILER(); ?>)

因此当全局搜索php_stream_open_wrapper被用来实现底层的PHP函数时也就不难发现还存在除对文件处理的函数之外其余的可以用来利用的函数,而getimagesize就是其中之一
